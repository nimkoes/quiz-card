# Quiz Card - 학습 카드 애플리케이션

## 개요

Quiz Card는 마크다운 파일로 작성한 학습 카드를 복습할 수 있는 웹 애플리케이션입니다. 간단한 문법으로 카드를 작성하고, 카테고리별로 관리하며, 즐겨찾기 기능을 통해 중요한 내용을 따로 모아 볼 수 있습니다. GitHub Pages를 통해 배포되어 어디서든 접근 가능하며, GitHub Gist API를 활용하여 즐겨찾기 정보를 여러 기기와 브라우저에서 동기화할 수 있습니다.

## 실행 방법

### 개발 환경 설정

1. 프로젝트 클론 또는 다운로드
2. 의존성 설치:
```bash
npm install
```

3. 개발 서버 실행:
```bash
npm run dev
```
개발 서버는 기본적으로 `http://localhost:5173`에서 실행됩니다.

### 빌드

프로덕션 빌드를 생성하려면:
```bash
npm run build
```
빌드 결과물은 `dist` 폴더에 생성됩니다.

### GitHub Pages 배포

이 프로젝트는 GitHub Actions를 사용하여 자동 배포됩니다. `main` 브랜치에 push하면 자동으로 빌드되어 GitHub Pages에 배포됩니다.

수동 배포가 필요한 경우:
```bash
npm run deploy
```

**주의사항**: GitHub Pages 설정에서 Source를 "GitHub Actions"로 설정해야 합니다. 저장소 설정 > Pages > Source에서 "GitHub Actions"를 선택하세요.

## 기능 설명

### 카드 관리
- 마크다운 파일에서 카드 자동 파싱: `public/content/` 폴더의 하위 폴더를 카테고리로 인식하고, 각 폴더 내의 `.md` 파일을 자동으로 스캔하여 카드를 추출합니다.
- 카테고리별 필터링: 여러 카테고리를 동시에 선택하거나 특정 카테고리만 선택할 수 있습니다.
- 전체 카드 관리: 모든 카드를 한 화면에서 검색, 필터링, 정렬하며 즐겨찾기를 일괄 등록/해제할 수 있습니다.

### 학습 모드
- 순서대로 보기: 카드를 파일 순서대로 표시합니다.
- 랜덤 보기: 카드 순서를 무작위로 섞어 표시하며, 새로고침 버튼으로 다시 섞을 수 있습니다.
- 즐겨찾기 필터: 즐겨찾기한 카드만 모아서 볼 수 있습니다.

### 즐겨찾기 기능
- 카드별 즐겨찾기: 각 카드에 별표 버튼을 클릭하여 즐겨찾기에 추가/제거할 수 있습니다.
- 크로스 디바이스 동기화: GitHub Gist API를 사용하여 즐겨찾기 정보를 클라우드에 저장하므로, 다른 기기나 브라우저에서도 동일한 즐겨찾기 목록을 볼 수 있습니다.
- 일괄 관리: 전체 카드 관리 화면에서 여러 카드를 선택하여 한 번에 즐겨찾기에 추가하거나 제거할 수 있습니다.

### 사용자 인터페이스
- 반응형 디자인: PC에서는 사이드바를, 모바일에서는 햄버거 메뉴를 사용하여 화면 크기에 맞게 최적화되었습니다.
- 포켓몬스터 테마: 어두운 배경과 밝은 텍스트로 구성된 게임 스타일의 테마를 적용하여 가독성을 높였습니다.
- 부드러운 애니메이션: 카드 전환 시 페이드 효과를 적용하여 자연스러운 사용자 경험을 제공합니다.

## 기술적으로 자세한 설명

### 아키텍처
이 애플리케이션은 React와 TypeScript로 작성되었으며, Vite를 빌드 도구로 사용합니다. 컴포넌트 기반 아키텍처를 채택하여 각 기능을 독립적인 컴포넌트로 분리했습니다. 상태 관리는 React의 기본 훅(useState, useEffect, useMemo)을 사용하며, 즐겨찾기 기능은 커스텀 훅(useFavorites)으로 캡슐화되어 있습니다.

### 데이터 파싱
마크다운 파일은 빌드 시점에 자동으로 스캔됩니다. `scripts/generate-content-index.js` 스크립트가 `public/content/` 폴더를 순회하여 카테고리와 파일 목록을 `index.json` 파일로 생성합니다. 런타임에는 이 JSON 파일을 읽어 카테고리 구조를 파악하고, 각 마크다운 파일을 요청 시점에 파싱합니다. 카드는 `<<<<<`와 `>>>>>` 구분자로 추출되며, `###` 이후의 내용은 추가 설명으로 분리됩니다.

### 즐겨찾기 동기화
즐겨찾기 정보는 GitHub Gist API를 통해 저장됩니다. 사용자가 GitHub Personal Access Token을 입력하면, 애플리케이션은 자동으로 Gist를 생성하거나 기존 Gist를 찾아 연결합니다. 즐겨찾기를 추가하거나 제거할 때마다 Gist의 내용이 업데이트되며, 다른 기기에서도 동일한 Gist를 읽어와 동기화됩니다. 토큰과 Gist ID는 브라우저의 localStorage에 저장되어 다음 접속 시 자동으로 로드됩니다.

### 반응형 디자인
Tailwind CSS의 반응형 유틸리티 클래스를 사용하여 화면 크기에 따라 레이아웃을 변경합니다. 768px 미만의 화면에서는 모바일 메뉴가 표시되고, 그 이상에서는 사이드바가 표시됩니다. 모바일 환경에서는 글씨 크기도 자동으로 축소되어 더 많은 정보를 화면에 표시할 수 있습니다.

### 빌드 프로세스
빌드 시점에 `generate-content-index.js` 스크립트가 먼저 실행되어 콘텐츠 인덱스를 생성합니다. 그 다음 TypeScript 컴파일러가 타입 체크를 수행하고, Vite가 React 컴포넌트를 번들링하여 정적 파일로 생성합니다. 생성된 파일은 GitHub Pages에 배포되어 정적 웹사이트로 서비스됩니다.

## 주요 용어 정의

### 카드 (Card)
마크다운 파일에서 `<<<<<`와 `>>>>>` 구분자로 감싸진 하나의 학습 단위입니다. 카드는 본문 내용과 선택적으로 추가 설명을 포함할 수 있으며, 각 카드는 고유한 ID를 가집니다. 카드 ID는 카테고리명, 파일명, 파일 내 인덱스를 조합하여 생성됩니다.

### 카테고리 (Category)
카드를 그룹화하는 단위로, `public/content/` 폴더의 하위 폴더 이름이 카테고리명이 됩니다. 사용자는 여러 카테고리를 동시에 선택하여 해당 카테고리들의 카드를 함께 볼 수 있습니다.

### 즐겨찾기 (Favorite)
사용자가 중요하다고 생각하는 카드를 표시하는 기능입니다. 즐겨찾기된 카드는 별표 아이콘이 채워진 상태로 표시되며, 즐겨찾기 필터를 사용하면 즐겨찾기된 카드만 모아서 볼 수 있습니다. 즐겨찾기 정보는 GitHub Gist에 저장되어 여러 기기에서 동기화됩니다.

### GitHub Gist
GitHub에서 제공하는 코드 스니펫 공유 서비스로, 이 애플리케이션에서는 즐겨찾기 정보를 저장하는 용도로 사용됩니다. JSON 형식으로 즐겨찾기된 카드 ID와 등록일시를 저장하며, GitHub Personal Access Token을 통해 접근 권한을 인증합니다.

### Personal Access Token (PAT)
GitHub API에 접근하기 위한 인증 토큰으로, 사용자가 GitHub에서 생성하여 애플리케이션에 입력합니다. 이 토큰은 `gist` 권한을 가져야 하며, 브라우저의 localStorage에 암호화되지 않은 상태로 저장됩니다. 토큰이 없으면 즐겨찾기 기능을 사용할 수 없습니다.
